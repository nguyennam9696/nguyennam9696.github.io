Ruby: Enumberable Method

Enumerable: What it is & How to use it

An Enumerable method in Ruby language is defined as a "mixin that provides a collection of classes with several traversal and searching methods, and with the ability to sort." - Ruby Doc Core. What does this all mean? Hmm.. Let's break things down to the human language. Let's begin with the term "mixin." For other languages, code sharing is done through the use of mixin between different classes. Imagine an English classroom. How is an English teacher to share his/her curriculum to other English teachers? In Ruby language, this curriculum can only be inherited from one teacher to another. However, with the use of Enumerables, the curriculum is allowed to be "mixed-in" or shared to another teacher across many different teachers if they like that English curriculum of course. Simple enough? It is.

Alright. That is mixin, but what about Enumerable? Enumerable is a module that can be mixed in to classes. Enumerable contains a BUNCH of methods that works with collection of code block. It is very useful to know Enumerable. It notably empowers Array, Hash, Set, Range, and File classes. Look at all of these Enumerable methods below.

In order for a class to use Enumerable, it must define an #each method. What the #each method does is that it passes each item in the collection to the code block.

Enumerable#map

The map method is one of the most important enumerable method in Ruby. Whatever the case is for enumerable, #map always returns an array. Awesome! It spits back out the same size as the original enumerable. 4 objects in the enumerable would equate to 4 objects in the array. #map method is different from #each method in the enumerable module because it maintains an accumular array of the results from the block. puts and map together will return nil, so becareful about that.

Enumerable#group_by

A #group_by method on an enumerable object takes the code block and returns a hash. Opposite from #map method because #map method returns an array. This code block executes for each of the object. Naturally, a hash has a key and a value. In this case, what would be our key and value? Our key would be the result of the hash. The value would be the array for all of the enumerable objects.

Enumerable#cycle

Just like your bicycle where your wheel spins round and round again and again, the #cycle method spits out the object over and over again in a loop. If you provide Ruby how many times you want to loop it, then it will loop for that many times only. If not, Ruby will loop forever.
